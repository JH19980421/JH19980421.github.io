### 다형성
<br>

한 객체가 여러가지 타입의 객체로 표현될 수 있는 능력을 말한다.  

자식 클래스가 부모 클래스를 상속 받고 있다고 가정 했을 때, 한 메소드의 매개변수가 부모 클래스로 정의되어 있는 경우, 자식 클래스도 매개변수로 지정할 수 있다.

부모 클래스는 자식 클래스의 생성자로 인스턴스를 만들 수 있지만, 자식 클래스의 메소드와 멤버변수는 사용할 수 없다. 또한, 자식 클래스는 부모 클래스의 생성자로 인스턴스를 만들 수 없다. 자식 클래스의 메소드와 멤버변수가 부모 클래스에는 없을 수 있기 때문이다.

<br>

### 메소드 오버라이딩
<br>

부모 클래스의 메소드가 자식 클래스에서 오버라이딩으로 재정의 되었을 경우, 자식 클래스의 메소드가 우선권을 가진다.

<br>

```java
class Exam {
    int score;
    int totalScore(Exam[] subjects) {
        int totalScore = 0;

        for (Exam subject : subjects) {
            totalScore+=subject.score;
        }

        return totalScore;
    }

    void printEncouragement() {
        System.out.println("시험 고생 많으셨습니다.");
    }
}

class History extends Exam {
    History(int score) {
        super.score=score;
    }

    @Override
    void printEncouragement() {
        System.out.println("역사 시험 고생 많으셨습니다.");
    }
}

class Literature extends Exam {
    Literature(int score) {
        super.score=score;
    }

    @Override
    void printEncouragement() {
        System.out.println("문학 시험 수고하셨습니다.");;
    }
}

class Math extends Exam {
    Math(int score) {
        super.score = score;
    }

    @Override
    void printEncouragement() {
        System.out.println("수학 시험 고생많으셨습니다.");
    }
}

class Science extends Exam {
    Science(int score) {
        super.score = score;
    }

    @Override
    void printEncouragement() {
        System.out.println("과학 시험 고생 많으셨습니다.");
    }
}

class ExamDay {
    public static void main(String[] args) {
        Exam[] student = {new History(40), new Literature(50), new Math(70), new Science(80)};
        Exam exam = new Exam();

        for(Exam subject : student) {
            subject.printEncouragement(); // 메소드 오버라이딩
        }

        System.out.println("총점 : " + exam.totalScore(student));
    }
}
```

### 추상 클래스

<br>

추상 클래스란 추상 메소드를 포함하는 미완성된 클래스이다.  추상 클래스를 사용하는 목적은 추상 클래스를 상속 받는 자손 클래스들이 공통적으로 가져야할 메소드들을 미리 정의하기 위함이다.  훗날에 각각 자손 클래스들에게 맞는 기능을 구현 함으로써  효율을 높일 수 있다.

접근 제어자를 별도로 지정하지 않는다면 private으로 생성된다. 생성자를 가질 수는 있지만, 인스턴스를 생성할 수는 없다.  오로지 상속을 통해서 자손 클래스들에 의해서만 완성될 수 있다. 

abstract 에는 static이 함께 사용될 수 없다. (static은 메모리가 로딩될때 완성되어야 하기 때문.)

<br>

```java
abstract class Unit {
    int x,y;

    void getPosition() {
        System.out.println("현재 위치는 (" + x + ", " + y + ") 입니다.");
    }

    abstract void move(int x, int y);
}
class Tank extends Unit{
    
    void move(int x, int y) {
        this.x += x/2;
        this.y += y/2;
    }
}
public class Marine extends Unit {
    
    void move(int x, int y) {
        this.x += 2*x;
        this.y += 2*y;
    }
}
import java.util.Random;

public class Field {
    public static void main(String[] args) {

        Random random = new Random();
        Marine marine = new Marine();
        Tank tank = new Tank();

        for (int i = 0; i < 3; i++) {
            marine.move(random.nextInt(10), random.nextInt(10));
            tank.move(random.nextInt(10), random.nextInt(10));
        }

        marine.getPosition(); // 현재 위치는 (12, 20) 입니다.
        tank.getPosition(); // 현재 위치는 (2, 6) 입니다.
    }
}
```

### 인터페이스

<br>

인터페이스란 오직 추상메소드와 상수만을 멤버로 가질 수 있는 기본 설계도.  인터페이스는 인터페이스만 상속받을 수 있으며, 하나의 클래스만 상속 받을 수 있는 클래스들의 특징과는 반대로 인터페이스는 여러 개를 상속 받을 수 있다.

- 인터페이스의 모든 멤버변수는 public static final 이다. (생략 가능)
- 모든 메소드는 public abstract 이다. (생략 가능)
- 다만 예외적으로 static, default 메소드를 사용할 수 있다.
- 인터페이스를 상속 받을 때는 implements 를 명시한다.
- 인터페이스의 메소드를 오버라이딩 할 때에는 부모 인터페이스 메소드의 접근 제어자 보다 더 넓은 범위를 지정해야한다.

또한, 인터페이스도 참조 변수를 통하여 인스턴스를 가질 수 있다. 인스턴스는 인터페이스를 상속받는 클래스로만 생성할 수 있다.

<br>

```java
interface Payment {
    void processPayment(int amount); 
    void cancelPayment();            
}

class KakaoPay implements Payment {
    @Override
    public void processPayment(int amount) {
        System.out.println("카카오페이 " + amount + "원 결제 승인");
    }

    @Override
    public void cancelPayment() {
        System.out.println("카카오페이 결제 취소");
    }
}

class CardPay implements Payment {
    @Override
    public void processPayment(int amount) {
        System.out.println("카드 " + amount + "원 결제 승인");
    }

    @Override
    public void cancelPayment() {
        System.out.println("카드 승인 취소");
    }
}

class BuySomething {
    public static void main(String[] args) {
        Payment kakao = new KakaoPay();
        kakao.processPayment(3000); // 카카오페이 3000원 결제 승인

        Payment card = new CardPay();
        card.processPayment(2000); // 카드 2000원 결제 승인
        
        kakao.cancelPayment(); // 카카오페이 결제 취소
        card.cancelPayment(); // 카드 승인 취소
    }
}
```
