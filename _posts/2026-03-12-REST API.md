### REST
<br>
REST(Representational State Transfer)란 WWW(World wide web)과 같은 분산 하이퍼미디어 시스템 아키텍처의 한 형식이다. 주고받는 자원에 이름을 규정하고, URL에 명시해 HTTP 메소드(GET, POST, PUT, DELETE)를 통해 해당 자원의 상태를 주고받는 것을 의미한다.
<br>
<br>

### REST API
<br>
REST API는 REST 아키텍처를 따르는 시스템, 혹은 애플리케이션 인터페이스이다. 아래는 REST API의 특징이다.

- Client - Server : 서버와 클라이언트는 서로 독립적이어야 한다.
    - 서로 독립적이라 서버 기능을 업그레이드해도 클라이언트를 새로 만들 필요가 없다.
- Stateless : 서버에 상태 정보를 따로 저장하지 않는다.
    - 서버가 클라이언트의 상태를 저장하지 않으니 서버 부하가 적고 확장이 쉽다.
- Cacheable : 한 번 받은 응답은 클라이언트가 저장해 두었다가 재사용할 수 있어야 한다.
    - 네트워크 트래픽을 줄이고 응답 속도를 높인다.
- Uniform Interface : 어떤 자원이든 HTTP 메서드, URI 규칙으로 접근해야 한다.
    - 프로그래밍 언어, 플랫폼 및 기술에 종속되지 않고 다른 언어, 플랫폼, 기술과 호환될 수 있다.
- Layered System : 클라이언트는 서버가 복잡도(여러 계층으로 이루어져 있는지)와 상관없이 서버와 연결되는 포인트만 알면 된다.
    - 보안을 강화하거나 서버 성능을 분산하기 좋다.
- Code on Demand : 서버가 클라이언트에 실행 가능한 코드(를 직접 보내서 기능을 확장할 수 있다.

### REST URL 설계 규칙
<br>

- URL의 마지막에는 ‘/’를 포함하지 않는다.
    - ex) http://localhost.com/user
- 언더바(_)가 아닌 하이픈(-)을 사용한다.
    - ex) http://localhost.com/user-account
- URL에는 행위(동사)가 아닌 결과(명사)를 포함한다.
    - ex) http://localhost.com/user/123
- URL은 소문자로 작성한다.
- 파일 확장자는 URL 에 포함하지 않는다.

### HTTP 메서드
<br>

GET (조회)

<br>

서버에서 데이터를 가져올 때 사용한다. 데이터를 변경하지 않으므로 여러 번 호출해도 결과가 같다.(idempotent, 멱등성)

<br>

```java
@RestController
@RequestMapping("/api/posts")
public class PostController {

    // 1. 전체 목록 조회
    @GetMapping
    public List<Post> getAllPosts() {
        return postService.findAll();
    }

    // 2. 특정 게시글 조회 (경로 변수 사용)
    @GetMapping("/{id}")
    public ResponseEntity<Post> getPostById(@PathVariable Long id) {
        Post post = postService.findById(id);
        return ResponseEntity.ok(post);
    }
}
```
<br>

- **`@GetMapping`**: HTTP GET 요청을 처리한다.
- **`@PathVariable`**: URL 경로에 포함된 값(id)을 변수로 가져온다.

<br>
<br>

POST (생성)

<br>

새로운 자원을 생성할 때 사용한다. 호출할 때마다 새로운 데이터가 쌓이므로 멱등성이 없다.

<br>

```java
@PostMapping
public ResponseEntity<Post> createPost(@RequestBody PostRequestDto requestDto) {
    Post savedPost = postService.save(requestDto);
    // 생성 성공 시 201 Created 응답과 함께 생성된 데이터 반환
    return new ResponseEntity<>(savedPost, HttpStatus.CREATED);
}
```
<br>

- **`@PostMapping`**: HTTP POST 요청을 처리한다.
- **`@RequestBody`**: 클라이언트가 보낸 JSON 데이터를 자바 객체(DTO)로 변환한다.

<br>
<br>

PUT(전체 수정)

<br>

자원의 전체를 업데이트한다. 만약 일부 필드를 누락하고 보내면 그 필드는 `null`이 될 위험이 있다.

<br>

```java
@PutMapping("/{id}")
public ResponseEntity<Post> updatePostFull(@PathVariable Long id, @RequestBody PostRequestDto requestDto) {
    Post updatedPost = postService.updateFull(id, requestDto);
    return ResponseEntity.ok(updatedPost);
}
```

<br>

PATCH(일부 수정)

<br>

자원의 특정 부분만 변경할 때 사용한다.

<br>

```java
@PatchMapping("/{id}")
public ResponseEntity<Post> updatePostPartial(@PathVariable Long id, @RequestBody Map<String, Object> updates) {
    Post updatedPost = postService.updatePartial(id, updates);
    return ResponseEntity.ok(updatedPost);
}
```
<br>
<br>


DELETE (삭제)

<br>

자원을 삭제할 때 사용한다.

<br>

```java
@DeleteMapping("/{id}")
public ResponseEntity<Void> deletePost(@PathVariable Long id) {
    postService.delete(id);
    // 삭제 후 보통 204 No Content를 반환.
    return ResponseEntity.noContent().build();
}
```

<br>

- **`@DeleteMapping`**: 특정 ID의 자원을 삭제한다. 삭제된 데이터를 다시 삭제 요청해도 서버 상태는 '삭제됨'으로 동일하므로 멱등성을 가진다.
