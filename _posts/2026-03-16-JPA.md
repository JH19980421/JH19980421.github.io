### ORM

<br>

ORM(Object Relational Mapping)은 객체 관계 매핑이다. 객체와 RDB(Relational Database)의 테이블을 자동으로 매핑하는 방법이다. 클래스는 데이터베이스의 테이블과 매핑하기 위해 만들어진 것이 아니기 때문에 RDB 테이블과의 불일치가 존재할 수 있는데, ORM은 이러한 사항을 해결하는 역할이다. SQL 쿼리를 직접 짜지 않아도 코드 상의 객체를 조작하는 것만으로 데이터를 관리할 수 있게 해준다.

<br>

- 생산성 및 코드 가독성 향상
    - 반복적인 CRUD(Create, Read, Update, Delete) SQL 작성이 대폭 줄어들어, 가독성이 좋아진다.
- 유지보수의 편의성
    - 데이터베이스 스키마가 변경되어도 객체 모델만 수정하면 된다.
- 데이터베이스 추상화와 독립성
    - 특정 데이터베이스(MySQL, PostgreSQL, Oracle 등)에 종속되지 않는다. 설정 파일에서 DB 종류만 바꾸면 쿼리 문법 차이를 ORM이 알아서 처리해주므로, DB 엔진을 교체하거나 멀티 DB를 지원할 때 매우 유리하다.
- 보안 및 실무 기능 제공
    - SQL Injection 방지 **:** 대부분의 ORM은 파라미터 바인딩을 기본으로 사용하여 보안 사고를 예방한다.
    - 캐싱 및 지연 로딩 **:**  성능 최적화를 위한 1차 캐시, 지연 로딩(Lazy Loading) 등의 고급 기능을 활용할 수 있다.
- 성능 오버헤드 (The N+1 Problem)
    - N+1 문제가 발생할 수 있다.

<br>

### JPA

<br>

JPA(Java Persistence API)는 자바 진영의 ORM(Object-Relational Mapping) 기술 표준이다. ORM의 개념을 자바 언어에서 실제로 구현하기 위해 만든  인터페이스의 집합이다. 데이터를 관리하는 일종의 가이드라인이고, 이 인터페이스를 구현한게 Hibernate이다.

<br>

객체는 상속 & 참조 중심이지만 DB는 외래 키에 중점을 두기에, JPA는 이 간극을 중간에서 메워주기에 SQL에 의존적임을 벗어나 객체 지향적인 코드를 작성할 수 있다.

<br>
<br>

### 영속성 컨텍스트(Persistence Context)

<br>

영속성 컨텍스트는  애플리케이션과 데이터베이스 사이에서 엔티티와 레코드의 괴리를 해소하는 기능과 객체의 저장소 역할을 한다.

<br>

- 1차 캐시 : 트랜잭션 내에서 동일한 데이터를 조회할 때 DB가 아닌 메모리에서 바로 가져와 성능을 높임.
- 동일성(Identity) 보장 **:** 동일한 트랜잭션 내에서 같은 ID로 조회한 객체는 항상 같은 인스턴스임을 보장.
- 쓰기 지연 (Transactional Write-behind)  **:** 커밋하기 전까지 SQL을 모았다가 한 번에 DB로 보냄.
- 변경 감지 (Dirty Checking) **:** 객체의 값만 수정하고 커밋하면, JPA가 변경 사항을 감지해 자동으로 UPDATE  쿼리를 실행.

<br>
<br>

### 엔티티의 생명주기 (Entity Lifecycle)

<br>

JPA에서 객체(Entity)는 영속성 컨텍스트와의 관계에 따라 4가지 상태를 가진다.

<br>

- **비영속 (New/Transient) :** 영속성 컨텍스트와 전혀 관계가 없는 상태 (그냥 자바 객체 생성 직후).
- **영속 (Managed) :** 영속성 컨텍스트에 저장되어 관리되는 상태.
- **준영속 (Detached) :** 영속성 컨텍스트에 저장되었다가 분리된 상태.
- **삭제 (Removed)  :** 삭제를 요청한 상태.
